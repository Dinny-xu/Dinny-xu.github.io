web项目部署

Spring Boot 瘦身部署策略

准备项目:

  • 自行准备一个Spring Boot 项目
  • 引入瘦身jar包相关pom.xml 依赖
  • 依赖中 configuration 是将已经瘦身过的项目依赖和项目jar文件上传到指定服务器中
  • 再执行打包之前配置一个Maven 启动命令

  • 配置clean package 一键清空打包

  • 执行clean package

  • 打包成功后本地jar包会自动上传到服务器并且自动启动项目
  • 以下为启动脚本 - 部分文件路径需自行更改

说明：

  • JAVA_BIN ：必须填写Java 绝对路径

Docker 部署

  • 服务器安装Docker 环境
    • 根据服务器内核可自行安装相应版本

方式一：

使用Docker 运行jar 文件

  • 切换root用户，在/etc/docker 目录下创建daemon.json文件，添加镜像源
  • 重启docker
  • 拉取Java镜像
  • 运行Jar文件
  • 将jar 包放入个人指定的目录，例如：/project/demo
  • 执行命令生成容器启动jar
  • -d :后台运行
  • -p :端口映射
  • -v :/project/demo:/project/demo:将宿主机的路径映射到容器中
  • -name :定义生成容器的名称

方式二：

  • 利用Docker创建镜像运行jar 文件
  • 上传jar包到指定的目录
  • 在该目录下创建Dockerfile文件
  • 将下面内容复制到Dockerfile中
  • FROM : 基础镜像，基于jdk8镜像开始
  • MAINTAINER : 作者
  • ADD : 将jar文件添加(复制)到镜像内，docker-demo.jar源文件，demo.jar复制得到的文件
  • COPY : 将应用的配置文件拷贝到镜像中
  • EXPOSE : 声明端口
  • ENTRYPOINT : docker启动时，运行的命令，这里容器启动时直接运行jar服务
  • 注意：Dockerfile文件与jar包在同一层目录，因为我的Dockerfile文件没有指定jar包目录

创建好Dockerfile文件之后，执行命令 构建镜像

  • demo : 镜像名称
  • test : 镜像的tag，如果不写默认latest
  • . :这个 . 一定不能忘记，最后的. 代表本次执行的上下文路径，ADD docker-demo.jar demo.jar ，中间的docker-demo.jar在哪里，就是根据这个 . 确定的，在上述Dockerfile，表示将当前文件夹下的docker-demo.jar和config 文件添加到容器中
  • 镜像构建成功后可查看镜像

运行镜像

  • 查看服务是否启动成功
  • 根据容器CONTAINER ID 查看服务日志
    • 1000 表示查看最后1000行

说明：

  • 由于Dockerfile 启动方式是将jar包打成镜像，所以每次更新Java代码的时候 ，都要先stop 正在运行的容器，将容器删除，再删除镜像，然后重新build 才能算是真正的重启
  • 写了一个自动化脚本，可以一步到位
  • 由于瘦身打包会在target 下生成一个thin的文件夹，将文件夹下root文件夹整体压缩上传到服务器
  • 在root 文件夹中 有config 、repository、Dockerfile、run.sh、demo.jar 5个文件

  • 其中Dockerfile 、run.sh 是自己创建的

  • config ：java 的配置文件

  • repository ：java 所有的依赖

  • Dockerfile ：构建脚本

  • run.sh : 自动化构建脚本

  • demo.jar : 压缩之后的代码

  • 每次更新pom。需要将repository压缩重新上传至服务器

  • 更新配置文件，需要将config重新上传至服务器

  • 更新代码只需要将jar 重新上传至服务器

  • 上传过后执行 run.sh 自动化脚本

  • 进入上传的root目录

  • 赋予执行权限
  • 启动脚本
  • 以下为执行过程 如果服务器带宽够高，1秒即可完成
